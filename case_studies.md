# SkillPulse Case Studies

These are representative personas, not real individuals. They are designed to show how SkillPulse can be used across common early-career scenarios.

## Case Study 1: Aarav, the Off-Campus Fresher

**Target role**

`Junior Data Analyst`, Pune

**Current profile**

- Python
- Pandas
- NumPy
- Matplotlib
- Advanced Excel

**Problem**

Aarav clears aptitude rounds but struggles in technical screening because his projects look academic and do not reflect business-facing reporting workflows.

**Market drift observed**

- Rising: `Power BI`, `SQL Window Functions`, `Dashboard Storytelling`
- Decaying: static charts and one-off CSV analysis as primary deliverables

**How SkillPulse helps**

- identifies that his analytical foundation is useful but incomplete
- surfaces the gap between notebook-style analysis and dashboard-oriented business reporting
- recommends a short roadmap focused on BI and SQL depth

**Suggested proof-of-skill**

`Real-Time Regional Sales Tracker`: an interactive sales dashboard using SQL-backed data and business KPIs

**Outcome**

Aarav moves from generic analysis projects to a portfolio artifact that better matches fresher analytics roles.

## Case Study 2: Priya, the Service-to-Product Aspirant

**Target role**

`Frontend Developer`, Bengaluru

**Current profile**

- HTML
- CSS
- JavaScript
- React class components
- Bootstrap

**Problem**

Priya’s take-home assignments are rejected because her stack and implementation style look outdated for modern startup engineering roles.

**Market drift observed**

- Rising: `TypeScript`, `Tailwind CSS`, `Next.js`, typed API integration
- Decaying: `Bootstrap`, class components, loosely typed frontend code

**How SkillPulse helps**

- reframes her issue as a market-alignment problem rather than a talent problem
- identifies missing high-signal tools and frontend architecture patterns
- suggests a compact proof-oriented migration plan

**Suggested proof-of-skill**

`Type-Safe Movie Explorer`: a React + TypeScript + Tailwind project with robust API handling

**Outcome**

Priya gets a clearer path to demonstrate modern frontend readiness through a recruiter-visible portfolio piece.

## Case Study 3: Rohan, the Hackathon Enthusiast

**Target role**

`AI/ML Intern`, Hyderabad

**Current profile**

- Python
- Scikit-learn
- TensorFlow basics
- Jupyter notebooks

**Problem**

Rohan’s portfolio includes standard academic ML projects that no longer stand out in a highly competitive internship market.

**Market drift observed**

- Rising: `Prompt Engineering`, `RAG`, `LLM API integration`, `LangChain`
- Decaying: common benchmark projects on sanitized datasets without deployment or product thinking

**How SkillPulse helps**

- preserves the value of Rohan’s core ML foundation while highlighting where he lacks competitive differentiation
- prioritizes applied generative AI and lightweight deployment
- recommends a proof project that feels current and demonstrable

**Suggested proof-of-skill**

`Campus Rulebook RAG Chatbot`: a document-grounded question-answering app with a simple UI

**Outcome**

Rohan shifts from generic ML demonstrations to an applied AI portfolio item aligned with current internship expectations.

## Case Study 4: Neha, the Techno-Functional Business Analyst

**Target role**

`Business Analyst`, Gurugram

**Current profile**

- Advanced Excel
- BRD/FRD writing
- SWOT analysis
- Basic Tableau

**Problem**

Neha struggles in interviews that involve engineering collaboration because she cannot independently inspect data or reason through API-driven workflows.

**Market drift observed**

- Rising: `SQL`, API literacy, JSON understanding, Agile workflow awareness
- Decaying: static spreadsheet-driven reporting without technical context

**How SkillPulse helps**

- identifies that her communication and requirements skills are useful but no longer sufficient alone
- maps the exact technical literacy gap needed for modern BA roles
- turns the gap into a short learning-and-proof sequence

**Suggested proof-of-skill**

`FinTech API Data Mapper & BRD`: a small portfolio piece combining API inspection, data mapping, and requirements writing

**Outcome**

Neha becomes better positioned for techno-functional roles that require both stakeholder communication and technical fluency.

## Cross-Case Insights

- Market value is increasingly tied to deployable, demonstrable work rather than isolated academic exercises.
- Static reporting is giving way to interactive dashboards, SQL-backed reporting, and explainable outputs.
- Foundational skills still matter, but they no longer differentiate candidates on their own.
- Technical literacy is rising even in roles that were previously considered mostly functional.
- Proof-oriented guidance is more actionable than generic course recommendation.
